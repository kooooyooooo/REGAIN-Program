# REGAIN Corebook

> スポーツ指導者向け AIアプリ開発コーチング

---

## 1. REGAINとは

**定義**

REGAINは、スポーツに関わる人（トレーナー・治療家・コーチ・ジム経営者など）が、AIを"道具"として使えるようになり、自分の現場課題を自分の言葉で言語化し、自分専用の制作物（資料・アプリ・文章・提案書・分析物）を最短距離で完成させるための**AIスクール × 伴走型コーチング事業**である。

**キャッチフレーズ**

「4週間で、現場が変わる。AIで武器を作る。」

**核心的価値**

「単にAIの使い方を教えること」ではなく、**「成果物が完成すること」**に価値を置いている。

## 3. 課題認識（Why REGAIN?）

- 世の中のAIスクールは"知識型"が多すぎる
- 動画を見て終わり、で何も残らない
- 現場の人ほど「作りたいもの」は明確なのに、言語化・設計・AI活用でつまずいている
- 忙しい現場人に「遠回りの学習」は耐えられない

**結論**: 「学ばせる」ではなく「一緒に作る」必要がある

---

## 4. ターゲット

### コアターゲット

- トレーナー（AT、S&Cコーチ）
- 治療家（整骨院・接骨院・理学療法士）
- トレーニングジム経営者／スタッフ
- 部活動・クラブチームの指導者
- アスリートを支える裏方全般

### サブターゲット

- 教員・部活動顧問（※リスクが高いため、現在は優先度低・情報収集段階）

### 共通のペインポイント

- 現場経験はある
- 勉強意欲は高い
- だが、時間も余裕もない
- IT／AIに苦手意識がある
- 事務作業（在庫管理、日報、コンディション記録）に忙殺されている
- 既存のSaaS（One Tap Sports等）は帯に短し襷に長しで、費用対効果を感じにくい

### ゲイン（得られるもの）

- 自分たちのチームにぴったり合ったツールを自作できる
- 業務効率化と市場価値向上（「AIが使えるトレーナー」）を実現

## 5. 提供価値（Value Proposition）

1. **安い**
2. **一般的なことは動画で完結**
3. **作りたい制作物を徹底的にヒアリング**
4. **マンツーマン or 少人数で伴走**
5. **無駄な学習ゼロ**
6. **時間効率が異常に高い**

**Key Value**: プログラミング教育（ティーチング）ではなく、AIエージェントを活用した「バイブコーディング（指示出し開発）」スキルの習得とマインドセット変革。現場で使えるアプリとその作り方を伝授する、いわば**「魚」と「釣り方」を両方伝授**する。

## 6. サービス基本構造

### フロー

```
① インプット（最低限）
    ↓
② ヒアリング（最重要）
    ↓
③ 伴走制作
    ↓
④ 成果物完成
```

### 各フェーズの内容

| フェーズ | 内容 |
|---------|------|
| インプット | AIの基礎、プロンプトの考え方、事例動画（短時間） |
| ヒアリング | 何を作りたいのか？誰のためか？どこで使うのか？いつまでに必要か？ |
| 伴走制作 | 一緒にプロンプトを作る、一緒にAIを叩く、一緒に修正する |
| 成果物完成 | 資料、文章、アプリ、業務フロー |

## 7. カリキュラム（4週間）

### Week 0：導入・商談

**目的**: 信頼構築と期待値調整、および成約

| 項目 | 内容 |
|------|------|
| アイスブレイク & ヒアリング | 簡易ヒアリングを行い、現状の課題を特定 |
| デモ & プレゼン | GenSparkの画面共有を行い、目の前でAIがアプリを作る様子を見せる |
| 期待値調整 | 「AIで楽に稼げる」等の誤解を解き、本事業の学習目的を再確認 |
| 契約・クロージング | クーリングオフ（1週間）、完全返金保証の説明 |

**成果物**: 契約書、ヒアリングスクリプト、イメージアップ用デモアプリ

### Week 1：成功体験と要件定義

**目的**: 「自分でも作れた」という感動体験と、AIへの指示（要件定義）の重要性を学ぶ

| 項目 | 内容 |
|------|------|
| 要求定義 | テンプレートから作りたいものを決定、Gemini等を活用し要求定義書を作成 |
| GenSpark活用 | AIエージェントへの指示の出し方をレクチャー |
| プロトタイプ確認 | 実際に動くアプリを見て「すげー」という成功体験を作る |

**成果物**: 完成イメージアプリ（プロトタイプ）×3種、要件定義書作成用プロンプト

### Week 2：改善サイクルとVibe Coding習得

**目的**: 自身のアイデアを形にするための「指示出し→修正」のサイクルを回せるようになる

| 項目 | 内容 |
|------|------|
| 要求定義書の作成 | 「AIデベロッパー」モードを使用し、より詳細なアプリ構築を指示 |
| 改善サイクル | Plan→Do→Check→Actionのサイクルを習得 |
| ティーチング | コンテキスト（文脈）の重要性、AIとの対話方法 |

**成果物**: プレビューアプリ、改善サイクル手順書、要件定義書

### Week 3：技術連携と本番デプロイ

**目的**: Web上に公開し、継続的に開発・運用できる技術基盤を整える

| 項目 | 内容 |
|------|------|
| 技術スタック連携 | GitHub, Vercel, Supabaseの連携設定 |
| 開発環境と本番環境 | developブランチとmainブランチの運用 |
| エラーマインドセット | エラーとの付き合い方を学ぶ |

**成果物**: デプロイ済みWebアプリ（開発環境/本番環境）

### Week 4：現場実装と運用計画

**目的**: 現場で実際に運用を開始するための準備とルールの策定

| 項目 | 内容 |
|------|------|
| 運用ルールの策定 | リンク共有方法、選手への説明、フィードバック収集方法 |
| リリース手順 | 本番環境への反映手順の最終確認 |
| 開発/修正フロー | develop環境で開発→テスト→本番反映のフロー徹底 |

**成果物**: リリース手順書、改善サイクル手順書、現場への導入計画

## 15. 本音・裏思想（暗黙知）

### ① 本当の敵

本当に戦っている相手は「AIリテラシー不足」ではない。

> **「現場で"こんなことできたらなぁ"を実現できるのに、解決策を考えきれていないまま、忙しさに埋もれている現場の人」**

REGAINは**AIスクールの皮を被った"強制力あるブートキャンプ"**

### ② ストイックな本質

表面上は「伴走」「ヒアリング」「やさしいAIコーチ」だが、内側はストイック。

- 「で、結局何を作りたいんですか？」
- 「それ、本当に現場で使います？」
- 「誰の時間を短縮したいんですか？」
- 「それ、今作らない理由あります？」

REGAINは**"逃げ道を全部潰す設計"** → だから成果物が出る

### ③ AIは媒介に過ぎない

興味があるのは：
- 思考が整理される瞬間
- 現場のモヤモヤが言語になる瞬間
- 「あ、これでいいんだ」と腹落ちする瞬間

### ④ 対象外の人

エンジニア、ガチのAIオタク、すでに自走できる人は対象外ではないが相性は微妙。

一番ハマるのは：
> 能力はあるし、現場経験もアイデアもある。でも「具現化」ができずに、しかもそれは「できないもの」と勝手に思っている人

**「才能が眠っている人を起こす事業」**

### ⑤ 教育事業ではない

- 教育 → 正解を教える
- REGAIN → 正解を一緒に作る

REGAINは**「制作型コンサル × コーチング」**に近い

### ⑥ 修了者の変化

REGAIN修了者は**資格ではなく"匂い"が変わる**
- 見た瞬間「この人、分かってる」
- 説明しなくても話が通じる
- 仕事の質が違う
