# REGAIN Agenda Creator

REGAINプログラムのWeekセッション用アジェンダを作成する。

## 前提確認

アジェンダ作成前に必ず確認:

1. **REGAIN_COREBOOK.md** - プログラム全体の思想・構造を把握
2. **前後のWeekとの繋がり** - 受講者がどの状態で臨み、どこへ向かうか
3. **Protocol/Wisdomの設計思想** - [references/design-philosophy.md](references/design-philosophy.md)を参照

## アジェンダの構造

```markdown
# {絵文字} Week{N}：{サブタイトル}

## 🏝️ 導入 (Check IN)
- 前Weekからの繋がり
- 今週の概要説明

## ✅ 宿題チェック
- 事前課題の確認項目

## 🎯 今日のゴール (Goal of the Day)
1. ゴール1（箇条書きで補足）
2. ゴール2
3. ゴール3

## 📋 セッションの流れ
### Phase N：{フェーズ名}
| 時間目安 | 内容 | 種別 |
|----------|------|------|
| XX分 | 内容 | Protocol/Wisdom |

## 📝 今日のポイント
- マインドセット
- 前Weekとの比較（該当する場合）

## 📚 教材
### Protocol（手順書）
- 🎨 REGAIN_Protocol {名前}

### Wisdom（知恵）
- ⛲️ REGAIN_Wisdom {名前}

## 📦 今日の成果物
- [ ] 成果物1
- [ ] 成果物2

## 🔗 次回予告：Week{N+1}
- 次週の概要
```

## 各Weekの位置づけ

| Week | テーマ | 受講者の状態変化 |
|------|--------|------------------|
| 0 | 導入・商談 | 興味 → 期待 |
| 1 | 成功体験と要件定義 | 不安 → 「できた！」の感動 |
| 2 | 改善サイクルとVibe Coding | 感動 → 「自分で直せる」自信 |
| 3 | 技術連携と本番デプロイ | 自信 → 技術的な安心感 |
| 4 | 現場実装と運用計画 | 安心 → 「現場で使える」確信 |

## 作成時の注意

1. **絵文字の使い分け**
   - Week見出し: 内容を象徴する絵文字（🔥🔄🚀📦など）
   - Protocol: 🎨
   - Wisdom: ⛲️

2. **トーンとマナー**
   - 「教える」ではなく「一緒に作る」
   - 専門用語は必ず解説を添える
   - 現場の言葉で書く

3. **時間配分**
   - 1セッション約60-90分を想定
   - Wisdomは「AIの待ち時間」に配置

4. **成果物は具体的に**
   - チェックボックス形式
   - 何をどう作成するか明記
